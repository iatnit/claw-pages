LLM
Large Language Model · 大语言模型
本质就是一个
「文字接龙游戏」
你给它开头，它猜下一个词
一个接一个往下写

GPT · Claude · Gemini
全都是这么干的
@运哥的AI笔记
02 / 09
🔤
Token
模型处理文本的最小单位
模型不认字！
要先把文字切成小块
1 Token ≈ 1.5-2个汉字

"程序员" → "程序" + "员"
"helpful" → "help" + "ful"

切法由模型自己学会
≠ 按词切分

💭
Context
上下文 · 模型的临时记忆
AI 的「工作记忆」
每次对话，程序会自动把
聊天历史 + 你的问题 + 系统规则
全部打包发给模型

所以它才能"记住"
你之前说过的话
@运哥的AI笔记
04 / 09
📐
Context Window
上下文窗口 · 记忆上限
记忆有容量限制！
主流模型：~100万 Token
≈ 150万汉字
≈ 一整套《哈利波特》📚

GPT-5.4 → 105万
Gemini 3.1 Pro → 100万
Claude Opus 4.6 → 100万

📝
Prompt
提示词 · 给AI的指令
你对AI说的话
就是 Prompt
User Prompt
→ 你输入的问题

System Prompt
→ 开发者偷偷设的「人设」
　你看不到，但一直在生效

说得越清楚 → 结果越好

06 / 09
🔧
Tool
工具 · AI的双手
光靠脑子不够
还得有「双手」
AI 自己查不了天气、发不了邮件

Tool = 给AI接上的外挂函数
查天气、搜店铺、读文件…

⚠️ 模型只发指令
真正调用工具的是平台
@运哥的AI笔记
07 / 09
🔌
MCP
Model Context Protocol · 统一接口
工具界的「Type-C」
以前：每个AI平台接口都不同
同一个工具要写3遍代码 😩

现在：MCP统一标准
写一次 → 所有AI都能用 ✅

就像所有手机统一用Type-C
@运哥的AI笔记
08 / 09
🤖
Agent
智能体 · 自主打工人
不用你一步步教
它自己想、自己干
自主规划 → 自主调工具 → 持续工作
直到任务完成 🎯

例：查天气 → 发现下雨
→ 自动搜附近雨伞店

Claude Code · Codex · Gemini CLI
@运哥的AI笔记
09 / 09
📋
Agent Skill
技能文档 · Agent的工作手册
提前写好SOP
Agent照着做就行
本质：一份 SKILL.md 文档

元数据层 → 名称 + 描述
指令层 → 目标 / 步骤 / 规则 / 格式

以后只需说「我要出门了」
Agent 自动查天气、列清单 ✅
